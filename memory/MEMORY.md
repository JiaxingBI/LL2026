# LaborLink_App Memory

## Project Overview
Power Apps custom code app (canvas app page) — factory workforce scheduling tool for LEGO manufacturing.
Built with React 19, TypeScript 5.9, Vite 7, @microsoft/power-apps SDK 0.3.21.
Bilingual EN/ZH. Deployed via `npm run build && pac code push`.

## Key Files
- `src/App.tsx` — tab router (attendance | assembly | employee | test)
- `src/components/AttendancePlan.tsx` — PRIMARY: ~1000+ line monolith. Full-year shift grid.
- `src/components/LaborScheduling.tsx` — assembly line worker allocation
- `src/components/EmployeeView.tsx` — employee kiosk self-service view
- `src/contexts/LanguageContext.tsx` — EN/ZH i18n context
- `src/data/dataverseLoader.ts` — data fetch + transform orchestration
- `src/hooks/useDataverseEmployees.ts` — Dataverse employee hook
- `src/hooks/useUserPhotos.ts` — batched Office 365 photo fetch
- `src/utils/attendanceValidation.ts` — Chinese labor law rules engine
- `src/generated/` — auto-generated by Power Apps CLI (models + services)
- `src/types/index.ts` — all domain type definitions
- `src/index.css` — Apple HIG-inspired CSS variables + utilities

## Tech Stack
- React 19.2 + TypeScript strict mode
- Vite 7.2 (base: "./"), port 3000
- @microsoft/power-apps 0.3.21 (SDK)
- lucide-react icons
- PostCSS + Autoprefixer
- NO Tailwind, NO Redux, NO React Query, NO router library
- Custom CSS variables (Apple-inspired): --bg-color #F5F5F7, --accent-blue #0071e3, --radius 12px

## Data Sources (Dataverse)
- jia_ll_demployees — employees (read)
- jia_ll_dshiftgroups — shift groups (read)
- jia_ll_dshiftplans — base schedule (read-only)
- jia_ll_fattendancereocrds — attendance exceptions (read+write)
- office365users — Office 365 connector (photos)

## Known Issues / Bugs
- attendanceValidation.ts uses local time instead of UTC+8 (will break for non-CN timezone users)
- attendanceRecords loaded from Dataverse but never merged back into displayed schedule
- gender field hardcoded to 'Male' for all employees
- isInitialized always true in App.tsx (never wired to SDK init)
- TestPage is dead code but shows as a nav tab
- Adjustment ID collision risk in handleShiftChange (uses Date.now() only)
- JSON.parse(JSON.stringify()) deep clone used for undo state (use structuredClone instead)

## Top Optimization Priorities
1. Virtualize the pivot table grid (react-window or @tanstack/virtual)
2. Split AttendancePlan.tsx into sub-components + custom hooks
3. Unify data fetching (AttendancePlan has inline fetch, others use useDataverseEmployees)
4. Fix timezone handling in validation
5. Replace alert() with custom alert modal in EmployeeView/AttendancePlan

## Design System
- Custom CSS vars in index.css — Apple HIG-inspired
- Components: .card, .btn .btn-primary/secondary/ghost, .badge, .input, .navbar, .nav-tabs, .modal-overlay, .modal-content
- Mix of className + inline style — inconsistent
- No dark mode, no focus-visible states, no skip-links (accessibility gaps)

## Dev Workflow
- `npm run dev` starts pac + vite together
- `npm run build && pac code push` to publish
- Power Apps CLI needed for data binding resolution
